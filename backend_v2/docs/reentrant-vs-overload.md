# 重入（Reentrant）与重载（Overload）的区别

## 基本定义

| 概念 | 中文 | 含义 |
|------|------|------|
| **重载 (Overload)** | 方法重载 | 同一个类中，方法名相同但参数列表不同（参数个数、类型或顺序不同），这是**编译时多态** |
| **重入 (Reentrant)** | 可重入 | 一个方法/锁可以被多个线程同时安全地调用，或者一个函数可以递归调用而不出问题，这是关于**线程安全**的概念 |

## 核心区别

- **重载是方法签名的问题** — 同一个名字，不同的参数列表
- **重入是线程安全的问题** — 能不能被重复/并发调用

## 重入的常见场景

- **可重入锁**：同一个线程可以多次获取同一把锁而不会死锁
- **可重入方法**：可以被递归调用，方法内部调用自己不会出问题
- **可重入函数**：在多线程环境下，多个线程同时调用同一个函数而不会产生竞态条件

## Agent 的不可重入性

在 Agent 实现中，状态检查是一个重要的工程习惯：

```java
Assert.notNull(userInput, "用户输入不能为空");

if (agentState != AgentState.IDLE) {
    throw new IllegalStateException("Agent 状态不是 IDLE");
}
```

Agent 具有内部状态（如 `IDLE`、`THINKING`、`EXECUTING`、`DONE`、`ERROR`），因此**不是可重入的**。如果 Agent 正在处理任务时又接收新任务，会导致状态混乱，所以必须检查状态是否为 `IDLE`，确保一次只处理一个任务。
